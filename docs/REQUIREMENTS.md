# Rensoo 要件定義書 (REQUIREMENTS)

- ステータス: ドラフト（STEP1 要件定義 / 全6フェーズ）
- 最終更新: 2026-06-26
- 次フェーズ: STEP2 設計（`/2-design`）

---

## 1. 背景・目的

アイデア発想・思考支援を目的とした Web アプリ。ユーザーが入力したキーワードを起点に、
LLM（生成AI）による連想語を**マインドマップ状**に可視化し、そこから**自走的に思考を広げる**体験を提供する。
ブレインストーミング・企画・命名・学習などの初動を、連想の連鎖で素早く加速することを狙う。

### コアバリュー（一言で）
> キーワードを入れて「作成」を押すだけで、連想がマインドマップとして自動で広がっていく。

---

## 2. 対象ユーザー

| ペルソナ | 状況 | 期待 |
|---|---|---|
| 企画・発想をしたい人 | アイデアの起点が欲しい | キーワードから連想を一気に広げたい |
| 学習・整理をしたい人 | 概念の関連を俯瞰したい | 関連語をマップで眺めて理解を深めたい |
| ライト利用者（ゲスト） | 登録せず試したい | ログイン不要でまず使いたい |
| 継続利用者 | 作ったマップを残したい | アカウントで保存・再編集したい |

---

## 3. ユーザーストーリー

- US-1: ユーザーとして、キーワードを入力し「作成」を押すと、連想語がマインドマップで表示されてほしい。
- US-2: ユーザーとして、自動でどんどん連想が広がる様子（自走展開）を見たい。
- US-3: ユーザーとして、自動展開と手動（クリックで展開）を切り替えたい。
- US-4: ユーザーとして、自動展開が無限に広がってコストが暴発しないよう、上限で止まってほしい。
- US-5: ユーザーとして、1ノードあたりに生成する連想語の件数を調整したい。
- US-6: ユーザーとして、ログインしなくてもとりあえず使ってみたい（ゲスト利用）。
- US-7: ユーザーとして、気に入ったマップを保存したいときにログインして保存したい。
- US-8: 継続利用者として、保存したマップを一覧から開いて再編集・削除したい。
- US-9: ユーザーとして、ノード（連想語）を手動で追加・編集・削除したい。

---

## 4. 機能要件

### 4.1 連想語生成（LLM連想語API）
- FR-1: キーワード（テキスト）を入力し「作成」ボタンで起点ノードを作り、初回の連想語を生成する。
- FR-2: 連想語の生成は **LLM（生成AI）** を用いる。入力語から連想される語を取得する。
- FR-3: 生成言語は **日本語**。
- FR-4: 1ノードあたりの生成件数は **可変**（ユーザーが調整可能。デフォルト値を設ける。例: 3〜10件、初期値6件）。
- FR-5: 生成結果は重複・空・無関係語をできる範囲で除去し、ノードとして整形して返す。
- FR-6: 生成中はローディング状態を示し、失敗時は日本語でエラーを表示し再試行できる。

### 4.2 マインドマップ表示
- FR-7: 起点ノードと連想語ノードを**ノード＋エッジ**でマインドマップとして描画する。
- FR-8: ズーム・パン（拡大縮小・移動）ができる。
- FR-9: ノード数が増えても破綻しにくいレイアウト（自動配置）を行う。

### 4.3 自走展開（自動連鎖／手動展開のトグル）
- FR-10: **自動連鎖展開**モード: 生成された連想語ノードから、さらに自動で連想を連鎖生成して広げる。
- FR-11: **手動展開**モード: ユーザーがノードをクリックしたときに、そのノードから次の連想を生成する。
- FR-12: 自動 / 手動を**トグルで切替**できる。
- FR-13: 自動連鎖の**停止条件**: 「**最大深さ**」と「**総ノード数の上限**」に達したら自動展開を停止する。
  - 既定値を設ける（例: 最大深さ3・総ノード上限50）。値は設計で確定。
- FR-14: 自動展開はいつでもユーザーが一時停止／停止できる。

### 4.4 ノードの手動編集（MVP対象）
- FR-15: ユーザーはノードを**手動で追加**できる。
- FR-16: ユーザーはノードのテキストを**編集**できる。
- FR-17: ユーザーはノードを**削除**できる（関連エッジも整合的に処理）。

### 4.5 アカウント・保存
- FR-18: **未認証（ゲスト）でも**キーワード入力〜連想生成〜マップ閲覧・操作ができる。
- FR-19: マップの**保存は認証必須**。ゲストが保存しようとした場合はログインを促す。
- FR-20: 認証は **OAuth**（外部プロバイダ。具体は技術選定で確定）。
- FR-21: ログインユーザーは保存したマップを**一覧**表示し、**開いて再編集**・**削除**できる。
- FR-22: 保存データはユーザー本人のみがアクセスできる（他人のマップは見えない）。

## 5. 非機能要件

### 5.1 性能
- NFR-1: 初回連想生成は通常時、体感で待てる範囲（目安: 数秒以内）に結果を返す。具体目標値は設計で確定。
- NFR-2: マインドマップは数十ノード規模でも滑らかに操作できる。

### 5.2 コスト・レート制御（重要）
- NFR-3: LLM呼び出しは自動展開の停止条件（深さ・ノード上限）でコスト上限を担保する。
- NFR-4: 過剰なAPI連打を防ぐ仕組み（多重実行抑制・必要に応じてレート制限）を設ける。

### 5.3 セキュリティ
- NFR-5: LLM APIキー等の機微情報はサーバー側で秘匿し、フロントに露出しない。
- NFR-6: 認証・認可を適切に行い、保存マップは所有者のみアクセス可能。
- NFR-7: 入力値の検証を行い、想定外入力で破綻しない。

### 5.4 可用性・運用
- NFR-8: LLM API障害時もアプリ自体は落ちず、エラー表示・再試行で回復できる。

### 5.5 対応環境
- NFR-9: モダンブラウザ（最新の Chrome / Edge / Safari / Firefox）で動作。
- NFR-10: PC を主対象とし、モバイル表示は閲覧・基本操作が破綻しない程度を最低ラインとする（最適化レベルは設計で確定）。

### 5.6 言語・UI
- NFR-11: UI・メッセージ・エラーは**日本語**。

## 6. スコープ

### 6.1 やること（MVP）
- キーワード入力 → 「作成」 → LLM連想語生成 → マインドマップ表示
- 自動連鎖展開 ⇄ 手動展開のトグル、停止条件（深さ＋ノード上限）
- 1ノードあたり生成件数の調整
- ノードの手動編集（追加・編集・削除）
- ゲスト利用可、保存は OAuth 認証必須、保存マップの一覧・再編集・削除

### 6.2 やらないこと（MVP対象外 / 後続候補）
- マップの画像エクスポート（PNG/SVG）
- 共有リンク（閲覧共有）
- ノードのメモ・色分け / カテゴリ
- リアルタイム共同編集
- 英語・多言語対応（当面は日本語のみ）

---

## 7. 前提・制約

- LLM を連想語ソースとして使用する（APIキー・利用コストが発生）。プロバイダは技術選定（STEP2）で確定。本環境的に Claude が第一候補。
- 永続データ（ユーザー・保存マップ）が必要なため、DB 設計（STEP2-④）を実施する。
- 認証は OAuth を採用。プロバイダ・実装手段は技術選定で確定。

## 8. 受け入れ条件（テスト可能）

- AC-1: キーワードを入力し「作成」を押すと、起点ノードと**1件以上**の日本語連想語ノードがマインドマップ上に表示される。
- AC-2: 生成件数設定を変更すると、次に生成されるノードの件数が設定値に追従する。
- AC-3: 自動展開モードでは、停止条件（最大深さ・総ノード上限）に達するまで連想が自動で連鎖し、達したら**自動的に停止**する。
- AC-4: 手動展開モードでは、ノードをクリックしたときにのみそのノードから連想が生成される（自動連鎖しない）。
- AC-5: トグル操作で自動／手動が切り替わり、切替が画面に反映される。
- AC-6: 自動展開中に停止操作をすると、それ以上ノードが増えない。
- AC-7: ノードを手動で追加・編集・削除でき、結果がマップに反映される（削除時に孤立エッジが残らない）。
- AC-8: 未ログイン（ゲスト）状態で、キーワード入力〜連想生成〜マップ操作が一通り行える。
- AC-9: ゲストが「保存」を実行するとログインが要求され、未ログインのままでは保存されない。
- AC-10: ログイン後にマップを保存でき、再ログイン後も一覧から開いて再編集・削除できる。
- AC-11: 他ユーザーの保存マップにはアクセスできない。
- AC-12: LLM 呼び出しが失敗した場合、日本語のエラーが表示され、再試行できる（アプリは落ちない）。
- AC-13: LLM APIキーがブラウザ側のネットワーク/ソースに露出しない。

### 8.1 UI 大規模改修（M6）の受け入れ条件（AC-14〜19）

> M6（TASKS の T17〜T22）で追加する UX 要件。既存 AC-1〜13 は縮退させない（機能は維持したまま画面/操作性を強化）。

- AC-14: アプリは**ホーム画面**と**マインドマップ編集画面**の2画面で構成され、相互に遷移できる（ホームの「作成」で編集画面へ遷移して生成が始まる）。
- AC-15: ホーム画面は、**未ログイン時は機能紹介**を表示し、**ログイン時は保存済みマインドマップの一覧**を表示する。いずれの状態でもキーワード入力＋作成ができる（ゲスト作成は AC-8 を維持）。
- AC-16: 編集画面のサイドメニューに、現在のマップのノードが**親子ツリー**で表示され、ツリーとキャンバスの選択が同期する。
- AC-17: マインドマップ生成中は**ノードのクリック操作が禁止**され、ローディング表示が出る（背景には生成中のマップがうっすら見える）。
- AC-18: 生成が終了すると、ノードをクリックして**編集・削除ができる**ようになる（生成中は不可・AC-7 の操作を状態でゲート）。
- AC-19: 生成終了時に、**マップ全体が収まるようズームが自動調整**される（fitView）。

## 9. 未決事項（設計フェーズへ持ち越し）

- 生成件数の既定値とレンジ（例: 初期6・範囲3〜10）。
- 自動展開の停止条件の既定値（最大深さ・総ノード上限の具体値）。
- 採用 LLM プロバイダ／モデル、呼び出し方式、出力フォーマット（→ TECH_STACK / FEASIBILITY）。
- OAuth プロバイダ（Google 等）と認証ライブラリ（→ TECH_STACK）。
- マインドマップ描画の採用ライブラリとレイアウトアルゴリズム（→ TECH_STACK）。
- 性能目標値（生成レイテンシ・操作 FPS の具体数値）。
- モバイル対応の到達レベル。
- データモデル（ユーザー・マップ・ノード・エッジ）の詳細（→ DATABASE）。
